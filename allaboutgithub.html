All About GitHub

Q. What is a Version Control System?
Tracks changes to files over time
Q. What is Git?
Git is a Version Control System that allows you to record, save, and update a codebases from a remote codebase providing a space for collaborators in program development.
Q. What is GitHub?
GitHub, Inc. is a platform and cloud-based service for software development and version control using Git
Q. What is the difference between git and GitHub?
GitHub is a hosting platform for git repositories while Git is simply said repository.
Q. Who started GitHub and how was it started?
Tom Preston-Werner and Chris Wanstrath created GitHub and it was started as a beta a few months before its official release in April 2008.

Git/GitHub Terms:

Repository
A repository is a central storage location where version-controlled files and the entire history of a project are stored.
Commit
Commit means saving changes to a file or repository
Fork
Forking means to create a personal digital copy of someone elses work to use as a resource for future programs
Push
Push refers to the action of uploading or sending local changes/commits from your local repository to a remote repository
Pull Request
A pull request means to make a reccommendation in updating a code in a codebase that can be accepted and merged into a codebase
Workflows
A workflow is a series of automated steps coded into a program
Issues
Issues refers to bugs disabling a languages ability to interpret code, needs to be addressed and reworked
Raw Button
Clicking the raw button in GitHub allows you to see the raw data in a program in text form
Blame Button
The blame button in GitHub allows you to see the commit history of a program
