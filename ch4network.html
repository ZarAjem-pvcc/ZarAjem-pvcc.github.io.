Networking and the Internet Terminology

  • Network Classifications
    1. LAN
      • LAN consists of a collection of computers in a single building or building complex.
    2. MAN
      • a MAN is a network of intermediate size, such as om spanning a local community.
    3. WAN
      • a WAN links machines over a greater distance perhaps in neighboring cities or on opposite sides of the world.
  • Network Models
    1. Bus Network
      • a topology where all machines are connected to a common communication line called a bus.
    2. Star Network-
      • a classification of network in which one machine serves as a central focal point to which all the others are connected
  • Protocals
    1. CSMA/CD Ethernet
      • Carrier Sense, Multiple Access with Collision Detection is a protocal that dictates that each message be broadcast to all machines on a network, detecting if transmissions clash. if two machines clash, both machines will pause then discontinue transmitting.
    2. CSMA/CA Wifi
      • Carrier Sense, Multiple Access with Collision Avoidance is a protocal designed to avoid transmission clashes between wireless machines by giving advantage to devices that have already been waiting for an opportunity to transmit.
  • Combining Networks
    1. Repeater
      • a device that passes signals back and forth between two bus networks
    2. Bridge
      • similar to a repeater but more complex, a bridge connects two buses, but it does not necessarily pass all messages across yhe connection. instead, it looks at the destination address that accompanies each message and forwards a message across the connection only when that message is destined for a computer on the other side.
    3. Switch
      • a switch is essentially a bridge with multiple connections, allowing it to connect several buses rather than just two. a switch produces a network consisting of several buses extending from the switch like spokes on a wheel.
    4. Router
      • routers are special purpose computer used for handling connection between networks to form an internet
    5. Gateway
      • a gateway is the "point" at which one network is link to an internet because it serves as a passagway between the network and the outside world
  • Client/Server Model
    1. P2P: Peer-To-Peer
      • P2P is a client/server model that involves processes that provide service to and recieve service from each other. P2P usually involves processes that execute on a temporary basis. P2P is essentially a system by which two processes communicate over a network.
    2. Distributed System
      • distributed systems consist of software units that execute as processes on different computers.
    3. Cloud Computing
      • cloud computing is when huge pools of shared computers on the network can be allocated for use by clients if as needed
  • Internet Addressing
    1. Tier 1
      • at the top of the ISP hierarchy are the few tier 1 ISPs consisting of very high speed, high capacity international WANs. these networks are the backbone of the internet and usually operated by large companies.
    2. Tier 2
      • connecting to the tier 1 ISPs are the tier 2 ISPs that tend to be more regional in scope and less potent in their capabilities (the distinction between tier 1 and tier 2 ISPs is often a matter of opinion). tier 2 ISPs are usually operated by companies in the communications business.
    3. Tier 3
      • tier 1 and tier 2 ISPs are essentially networks of routers that collectively provide the internet's communication infrastructure. the many possible pathways between routers gives this core of the internet redundancy, meaning that most or all of the internet remains connected even when very important tier 1 routers fail. access to this core is usually provided by an intermediary called an access or a tier 3 ISP.
  • Ip Address
    1. ICANN - webpage
      • Internet Corporation for Assigned Names and Numbers is a nonprofit organization established to coordinate the internet's operation. blocks of consecutively numbered ip addresses are awarded into ISPs by the ICANN.
    2. DNS
      • a Domain Name System is a collective of servers that convert a messages' addresses into an IP addresses before transmitting.
    3. TLD: Top-Level Domain
      • TLDs are suffixes folling the period at the end of a domain used to classify said domain. for example, google.com, the ".com" is the TLD. some other examples of TLDs are .edu, .gov, .org, etc.
    5. Country-Code TLDs - webpage
      • country cold tlds are tlds used to catagorize specific official domains for countries. and example of this would be .au for austrailia.
  • Internet Protocals
    1. FTP
      • a File Transfer Protocal or FTP is a application for accessing another computer from a great distance.
    2. HTTP
      • the Hyper Text Transfer Protocal is a powerful protocal allowing for a network application to handled by a webpage.
    3. SMTP
      • the Simple Mail Transfer Protocal defines a way that two computers on the network may interact when transmitting an email message from one host to the other.
  • World Wide Web
    1. URL
      • In order to locate and retrieve documents on the web, each document is given a unique address called a Uniform Resource Locator or URL. each URL contains the information needed by a browser to contact the proper server and request the desired document.
    2. Client-Side and Server-Side Activities
      • in the instance that a client accesses a webpage involving animation or one that allows a customer to fill out an order and from and submit the order these would require additional activity from either the browser or the webserver. such activities are called client side activities if they are performed by a client like a browser, or server-side activities if they are performed by a server like a webserver.
  • Internet Four-Layer Software
    1. Application Layer
      • a layer in software where messages originate before they are sent to the next 3 layers. after the message is processed it is shot back up the heirarchy to the application layer
    2. Transport Layer
      • the transport layer is the second layer down in the heirarchy and it is responsible for chopping the message from the application layer into packets
    3. Network Layer
      • the network layer is the third layer down in the software's layer heirarchy. the network layer is responsible for assigning addresses for each of the packets provided by the transport layer
    4. Link Layer
      • the link layer is the fourth and last layer in the softwares layer hierarchy and it is responsible for transferring the packets
  • Internet Is:
    1. Packet-Switching Network
      • the internet is a packet-switching network through which digital data is sent by breaking the data into blocks of bits called packets, which contain both the data being transmitted and the control information for routing the data.
    2. Fault Tolerant
      • the internet is fault tolerant in the sense that the internet will continue to function in the occurance that there is some sort of failure withing the processes making up networks etc.
    3. Redundant
      • the internet is redundant in the sense that all pathways on the internet are interconnected
  • Cybersecurity
    1. Malware
      • malicious software that could be transferred to, and executed on, the computer itself, or it might attack the computer from a distance. examples of these softwares include, viruses, worms, trojan horses, and spyware.
    2. Virus
      • a virus is a software that infects a computer by inseting itself into programs that already reside in the machine. then when the "host" program is executed, the virus is also executed. when executed, many viruses do little more than try to transfer themselves onto other programs withing the computer but some can cause mass data damage and corrupt important programs on your computer.
    3. Worm
      • a worm is an autonomous program that transfers itself through a network, taking up residencei n computers and forwarding copies of itself to other computers.
    4. Trojan Horse
      • a trojan horse is a program that enter the computer system disguised as a desirable program, such as a game or a useful utility package, that is willingly imported by the victim. once in the computer, the trojan horse performs additional activities that might have harmful effects.
    5. Spyware
      • spyware is a software that collects information about activities at the computer on which it resides and reports that information back to the instigator of the attack. some companies use spyware as a means of building customer profiles, and in this context it has questionable ethical merit.
    6. Phishing
      • phishing is a technique of gaining information malicious instagators use by explicitly by simply asking for it.
    7. DDoS Attack
      • a denial of service attack or DoS attack is the process of overloading a computer with messages. dos attacks can disrupt a company's business and in some cases bring all of a company's commercial activity to a halt.
  • Protection
    1. Firewall
      • firewalls are a filter installed at a point where data is flowing to dictate which data is allowed to pass through and which is blocked. and example of this would be a firewall being installed at the gateway of an organization's intranet to filter messages passing in and out of the region.
    2. Spam Filter
      • spam filters are firewalls designed to block unwanted email using sophisticated to destiguish between desirable email and spam.
    3. Proxy Server
      • proxy servers are software units that acts as an intermediary between a client and a server with the goal of shielding the client from adverse actions of the server.
    4. Antivirus Software
      • antivirus software is used to detect and remove the presence of unknown viruses and other infections. usually antiviruse softwares can be catagorized as to what maliciously integrated software they are protecting against.
    5. Cryptography
      • cryptography is the study of sending and recieving messages securely in the presence of adversaries.
    6. Public Key Encryption
      • public key encryption involves techniques by which the encryption and decryption algorithms are known, and one of the keys is known, but the encoded message remains secure.
