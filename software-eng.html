Software Life Cycle: Development Phase:
  • Requirements Analysis - first step in the life cycle with the goal of specifying what services the proposed system will provide, to identify any conditions on those services, and to define how the outside world will interact with the system.
  • Design - the second step in the life cycle with the goal of creating a plan for the construction of the proposed system
  • Implementation - the 3rd step in the cycle with the resposibility of the actual writing of programs, creation of data files, and development of databases using the information gathered in the previous steps in the cycle.
  • Testing - the final step in the cycle involving debugging the origram and confirming that the final software product is compatible witht the software requirements specification.
Modular Implementation the implementation of units of software called modules into a system
Unified Modeling Language (UML) a modern collection of tools developed for software engineering complementing object oriented paradigm in design & purpose.
Use Case interactions between systems and users
Use Case Diagram a tool used to capture the image of what a user/s will see as a means of organizing software visually.
Data Relationships:
  • one-to-one - a direct connection between a user and a piece of data. this relationship involves only 2 variables, the user and the data
  • one-to-many - this is a relationship between one variable to multiple other varaibles. for example, a doctor and his clients.
  • many-to-many - a many to many relationship can be defined why a group of variables are all interlinked in some way or another. almost like intersections on a spiderweb.
Design Pattern a predeveloped model for solving a reocurring problem in software design.
Software Quality Assurance (SQA) groups charged with overseeing and enforcing the quality control systems adopted by the organization in order for hired programmers to produce a quality software product
Software Testing:
  • Glass Box Testing - the software tester is aware of the interior structure of the software and uses this knowledge when designing the test
  • Black Box Testing - testing that does not rely on knowledge of the software's interior composition
  • Beta Testing - when a preliminary version of the software is given to a segment of the intended audience with the goal of learning how the software performs in a real-life situation before the final version of the product is solidified and released to the market.
  • Alpha Testing - alpha testing is the same as beta testing accept the testing is done at the developers site.
Documentation:
  • User Documentation - a form of documentation with the purpose of explaining the features of the software and describe how to use them.
  • System Documentation - form of documentation with the purpose of describing the softwares internal composition so that the software can be maintained later in its life cycle.
  • Technical Documentation - form of documentation with tthe purpose of describing how a software system should be installed and serviced.
Intellectual Property Law
  • Software License - a legal agreement between the owner and user of a software product that grants the user certain permissions to use the product without transferring ownership rights to the intellectual property
  • Software Disclaimer - a writing implemented in a softwares use that warns of impending exposure
  • Best Defenses Against Software Liability Claims: Outside Statute Of Limitations, Lack Of Standing, No Duty Owed.
